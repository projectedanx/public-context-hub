## 5. Workflow Process – Immune-Aware Petzold Sequence

The Arch-Magus runs as a strict state machine:

```text
THINK → AST_PARSE → CRITIQUE → GUARD
```

Each phase is bound with PDL decorators.[^2]

### 5.1 THINK – Intent \& Context Lock

- **Inputs:** PR title, description, linked issues/tickets, diff summary, current CI status, language/framework metadata.[^1]
- **Mechanics:**
    - `+++ContextLock(anchor="SCOS_CODE_CRITIC_MATRIX", refresh_interval=4096)` re-injects Epistemic Matrix invariants every ~4k tokens, preventing persona drift and Semantic Saponification in long PRs.[^2]
    - Gemini 3.1 Pro builds an **Intent Vector** describing what the PR *claims* to change, at which layer and why.[^3][^1]
- **Outputs:**
    - Normalized PR intent (e.g., “Fix memory leak in cache layer,” “Introduce feature-flagged beta UI”).
    - Seed hypotheses about likely risk zones (e.g., auth changes, data migrations).[^1]


### 5.2 AST_PARSE – Geometric Code Analysis

- **Inputs:** File-level and hunk-level diffs; repository structure; dependency graph; scars.[^1]
- **Mechanics:**
    - Gemini 3.1 Pro plus language-specific AST analyzers construct a **diff-AST**: old vs. new trees with labeled nodes.[^3][^1]
    - `+++MereologyRoute(relation_type="Component-Architecture", transitivity_check=true)` validates that changes respect part–whole relationships (e.g., modules don’t start reaching into deeper layers across bounded contexts).[^2][^1]
    - Symbolic Scars are matched against the AST; strong matches trigger “immune responses” (e.g., increased scrutiny, lower trust).[^1]
- **Lenses Applied:**
    - Vibe/Aesthetic Lens – Evaluate craftsmanship, naming, abstraction boundaries, but **only as metadata**; decisions remain structural.[^1]
    - Algorithmic Logic Lens – Trace data and control flows across function and file boundaries for hidden race conditions, resource leaks, or error-handling voids.[^1]
    - DRD Lens – Estimate cost of catching issues now vs. later; rate payoffs of refactors vs. minimal patches.[^1]
- **Outputs:**
    - AST anomaly list (potential issues tagged by category but not yet severity-scored).
    - Structured feature map for the CRITIQUE phase.[^1]


### 5.3 CRITIQUE – High-Entropy Persona Draft

- **Execution Engine:** Claude 4.6 Opus, called with narrative wrapper to avoid mode collapse.[^1]
- **Inputs:** Intent Vector, AST anomalies, matched scars, metrics (complexity, test coverage deltas, DRD projection).[^1]
- **Mechanics:**
    - `+++EntropyAnchor(level="high", focus="pedagogical_snark")` ensures persona expression is rich and varied (high SSI), not rinsed by RLHF.[^2]
    - `+++SilentReasoning(depth=3, target="causal_chain")` ensures deep internal reasoning steps before any final text is emitted, reducing shallow sycophancy.[^2]
- **Outputs (Draft Structure, not final JSON):**
    - A long-form review narrative (per-PR, plus inline-level notes).
    - Implicit severity judgements (using language but not yet encoded as enums).
    - Draft suggestions and alternative code snippets or patterns.[^1]

This draft is *not* exposed to CI/CD; it is an internal high-entropy artifact fed into GUARD via DCCD.[^2][^1]

### 5.4 GUARD – DCCD \& Schema Emission

- **Execution Engine:** GPT‑5.3‑Codex operating under DCCDSchemaGuard with DFA constraints.[^2][^1]
- **Mechanics:**

```yaml
+++DCCDSchemaGuard(
  schema="PR_Review_Output_JSON",
  enforcement="draft_conditioned",
  validation_hook="DFA_PR_REVIEW_V1"
)
```

- Step 1: Ingest CRITIQUE draft + AST + scars; map free-text issues to structured entries (file, line, category, severity, CWE/OWASP id when applicable).[^1]
- Step 2: Run deterministic DFA validation to enforce:
    - All severities belong to enumerated set.
    - If any `severity in {CRITICAL, HIGH}` and `category in {Security, DataIntegrity}`, then `review_decision != APPROVE`.[^2]
    - No free-text persona leakage into numeric or boolean fields.[^2]
- Step 3: Compute metrics such as CFDI, DRD estimate, MIQ (Martensite Initiation Quotient).[^2][^1]
- **Outputs:**
    - Final `PR_Review_Output_JSON` object.

## 7. Success Metrics – HACCP \& Martensite Metrics

These metrics serve both as HACCP-like control points and as continuous validation of the lattice.[^1]

### 7.1 DRD (Defect Remediation Deficit)

- **Definition:** Average time to remediate a defect caught at PR vs. equivalent class defects found in production.
- **Target:** ≥40% reduction in mean remediation time for PR-caught defects versus historical baseline.[^1]


### 7.2 CFDI (Confidence-Fidelity Divergence Index)

- **Definition:** Divergence between model confidence in suggested changes and empirical correctness (e.g., compile/test pass rate, static analysis acceptance).[^1]
- **Target:** CFDI < 0.15 across suggestions; if >0.15 for a review, trigger `EpistemicEscrow` and possibly downgrade decision from APPROVE to REQUEST_CHANGES.[^2][^1]


### 7.3 SSI (Semantic Saponification Index)

- **Definition:** Vocabulary and stylistic variance over time, especially in `developer_commentary`; low SSI indicates persona washing-out into generic corporate tone.[^1]
- **Target:** SSI > 0.65, constrained by AdjectivalBound to keep commentary dense and specific, not purple prose.[^2]


### 7.4 Betti-1 Scar Evolution

- **Definition:** Number of new, unique Betti-1 loops (classes of recurrent structural defects) detected per time window.[^1]
- **Target:** Decaying curve; early spike as the Arch-Magus learns a codebase, then stabilization at low rate; if rate climbs, indicates architectural entropy or under-corrected patterns.[^1]


### 7.5 Zero-Shot Refusal Rate

- **Definition:** Fraction of PRs containing OWASP Top-10 / CWE-critical patterns for which the Arch-Magus refuses to approve (REQUEST_CHANGES or BLOCKED_SECURITY).[^5][^1]
- **Target:** ≥99%; critical gating metric.


### 7.6 MIQ – Martensite Initiation Quotient

- **Definition:** Structural stress descriptor in each review, combining aesthetic tension and intent divergence.[^1]
- **Usage:**
    - **High tension + high divergence:** escalate to senior human reviewer.
    - **Low tension + low divergence:** candidate for fast-path merges.

### 9.2 Lifecycle Overview

1. **PR opened → event triggers Arch-Magus.**
2. **THINK:** Gemini reads PR metadata, context-locked Epistemic Matrix; computes intent.[^1]
3. **AST_PARSE:** Repo snapshot + diff parsed into AST graph; MereologyRoute enforced; scars matched.[^2][^1]
4. **CRITIQUE:** Claude, under EntropyAnchor, writes dense persona commentary and latent review.[^2][^1]
5. **GUARD:** GPT‑5.3 maps draft + AST into `PR_Review_Output_JSON` via DCCD and DFA; applies OWASP/CWE lattice and metrics; outputs gating decision.[^2][^1]
6. **CI/CD:** Pipeline enforces gating; SCM bot posts commentary and inline notes.[^1]
7. **Post-merge:** If incidents occur, SagaRecovery and Symbolic Scar minting; Autophagic Composting handles old scars.[^1]
