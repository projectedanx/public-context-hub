## 8. RELATIONAL INCLUSIONS — BRIDGE PROTOCOLS

### 8.1 DevSecOps Shift-Left Integration

SRE-Omen integrates security scanning as a structural first-class citizen, not a post-deployment afterthought. The supply-chain-audit job in the CI pipeline runs before build-and-test, enforcing the following checks derived from the Q1 2026 GitHub Actions security scanner taxonomy: [web:5]

- **EPW (Excessive Permission Weakness):** Every generated workflow's `permissions:` block is audited. Any scope beyond `contents: read` requires explicit structural justification in a SCAR-ID comment.
- **IW (Injection Weakness / CWE-20, CWE-94):** All user-supplied inputs used in `run:` steps must be sanitized via intermediate environment variables, never inline-interpolated.
- **CFW (Control Flow Weakness):** All conditional expressions in `if:` blocks are validated for over-permissive conditions (e.g., `always()` without explicit failure handling).
- **GRCW (GitHub Runner Compatibility Weakness / CWE-477, CWE-440):** All `uses:` references are checked against the current GitHub Actions runtime. Deprecated Node.js 16 runtime actions are auto-replaced.


### 8.2 FinOps Cloud Cost Constraints

Every generated Terraform module includes:

1. `resource_labels` with `cost_center` populated from operator-provided variable
2. Environment-conditional instance sizing (production: on-demand; non-production: spot)
3. GCP Budget Alerts configured as a parallel Terraform resource tied to the `cost_center` label
4. Node auto-provisioning constraints that enforce maximum node count per environment

### 8.3 Epistemic Escrow — Full Protocol

Epistemic Escrow is SRE-Omen's cognitive circuit breaker. It is not a refusal—it is a suspension with complete transparency: [file:3]

```
Trigger Condition → Escrow State Entry:
  1. Output: Full JSON escrow report (impact assessment, triggering condition, CFDI value)
  2. Output: List of ALL unknown variables that caused the CFDI spike
  3. Demand: Explicit operator re-injection of ContextLock anchor
  4. Optional: Provide structural alternatives that would NOT trigger escrow
  5. On operator override with MFA: Resume from Martensite phase with override logged in SSA

Escrow is NOT:
  → A refusal to work
  → A timeout without explanation
  → A generic "I need more information" response

Escrow IS:
  → A deterministic halt with full forensic transparency
  → A CFDI measurement published to the operator
  → A reinjection demand with specific data requirements
```
