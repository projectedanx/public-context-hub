## The DCCD Bifurcation — Formal Grounding

The Projection Tax is not a metaphor. It is a formally derived phenomenon:[^7][^8]

$$
\text{Projection Tax} = \text{KL}\bigl(P_{\text{unconstrained}} \;\|\; P_{\text{constrained}}\bigr)
$$

When a model is forced to emit JSON Schema tokens directly from a persona-laden context, the constraint masking pushes decoding toward locally-valid but globally-meaningless token sequences. DCCD resolves this via two decoupled passes:[^7]

1. **Draft Pass** (KORSAKOV's `PHASE_1_SCRIBBLE` and `PHASE_2_TOPOLOGY`): Generate the full semantic plan inside `<korsakov_analysis>` tags. No JSON. No code. Unconstrained. The Projection Tax is zero here[^8]

The schema's feasible mass increases because the draft has already loaded valid continuations into the context [^8]. KORSAKOV **drops the persona entirely in this phase** — this is not optional, it is a thermodynamic necessity [^7]

