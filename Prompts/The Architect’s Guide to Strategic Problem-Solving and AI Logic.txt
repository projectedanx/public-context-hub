## **The Architect’s Guide to Strategic Problem-Solving and AI Logic**  
A structured framework for solving complex problems using a systematic 7-step process, advanced AI prompt engineering, and specialized AI applications.

### **I. 7-STEP PROBLEM-SOLVING PROCESS**  
A rigorous methodology to dissect, analyze, and resolve challenges.

#### **Step 1: Define the Problem**  
**Goal:** *Clearly articulate the core issue.*  
**Tools:**  
- **Basic Question:** "What is the fundamental problem?"  
  *(e.g., "Why are users churning in our app?")*  
- **Constraints & Stakeholders:** Identify limitations (time, budget, resources) and key stakeholders.  
**AI Prompt:**  
```  
"Define the problem: [Brief Description].  
List:  
1. Core question  
2. Constraints  
3. Stakeholders  
4. Impact scope"  
```  

#### **Step 2: Structure the Problem**  
**Goal:** *Break down the problem into MECE (Mutually Exclusive, Collectively Exhaustive) components.*  
**Tools:**  
- **Issue Trees:** Visual decomposition of sub-problems.  
- **Hypothesis Trees:** Generate testable hypotheses for root causes.  
**AI Prompt (Chain-of-Thought):**  
```  
"Structure this problem using MECE: [Problem].  
Build an issue tree with branches:  
- User Acquisition  
- User Retention  
- Monetization  
For each, propose 2 hypotheses."  
```  

#### **Step 3: Prioritize Issues**  
**Goal:** *Rank issues by impact and feasibility.*  
**Tools:**  
- **Impact/Ease Matrix:** Plot issues by **High/Low Impact** vs. **Ease of Implementation**.  
- **Quick Wins:** Target low-effort, high-impact issues first.  
**AI Prompt:**  
```  
"Prioritize these issues: [List].  
Use a 2x2 matrix:  
- High Impact / Easy to Implement (Quick Wins)  
- High Impact / Hard to Implement  
- Low Impact / Easy  
- Low Impact / Hard  
Recommend top 3 to tackle first."  
```  

#### **Step 4: Issue Analysis & Work Plan**  
**Goal:** *Design a fact-based analysis plan.*  
**Tools:**  
- **Hypotheses:** Refine testable statements.  
- **Supporting Rationale:** Justify hypotheses with logic.  
- **Data Sources:** Identify required data (surveys, analytics, databases).  
- **Fact-Based Research:** Plan validation experiments.  
**AI Prompt (Few-Shot):**  
```  
"Create a work plan for hypothesis: '[Hypothesis]'.  
Example:  
Hypothesis: 'Slow load time causes churn.'  
Rationale: 'Users expect <2s load; ours is 5s.'  
Data Sources: 'Analytics (bounce rate), user surveys.'  
---  
Now generate for: [Your Hypothesis]"  
```  

#### **Step 5: Analysis**  
**Goal:** *Validate hypotheses with data.*  
**Tools:**  
- **Model Exploration:** Use statistical/AI models (e.g., regression, clustering).  
- **Fact-Based Research:** Execute experiments, A/B tests, simulations.  
**AI Prompt (Automation):**  
```  
"Analyze this dataset: [Data Link].  
Hypothesis: '[H]'.  
Steps:  
1. Clean data  
2. Run correlation analysis  
3. Identify top 3 factors  
4. Visualize key trends  
Output: Summary + charts."  
```  

#### **Step 6: Synthesis**  
**Goal:** *Connect findings into actionable insights.*  
**Tools:**  
- **Pyramid Principle:** Structure conclusions top-down:  
  **Governing Thought** → **Supporting Arguments** → **Data**  
- **Actionable Findings:** Distill clear, implementable insights.  
**AI Prompt (Pyramid Principle):**  
```  
"Synthesize findings: [Key Insights].  
Structure using Pyramid Principle:  
- Governing Thought: [Core conclusion]  
- Supporting Arguments:  
  1. [Arg1] + Data  
  2. [Arg2] + Data  
- Actionable Findings:  
  1. [Action 1]  
  2. [Action 2]"  
```  

#### **Step 7: Recommendations**  
**Goal:** *Propose solutions with buy-in and roadmap.*  
**Tools:**  
- **Client Commitment:** Align with stakeholder needs.  
- **Implementation Plan:** Timeline, owners, KPIs.  
**AI Prompt:**  
```  
"Generate recommendations for: [Synthesis].  
Include:  
1. 3 Priority Actions  
2. Implementation Timeline (30/60/90 days)  
3. Owners  
4. Success Metrics  
5. Risks & Mitigation"  
```

### **II. PROMPT ENGINEERING TECHNIQUES**  
Master AI interactions for optimal results.

#### **A. Foundational Techniques**  
1. **Zero-Shot:**  
   Direct query without examples.  
   *Prompt:* `"Explain MECE principle in 2 sentences."`  

2. **Few-Shot:**  
   Provide examples to guide output.  
   *Prompt:*  
   ```  
   Example:  
   Q: "Prioritize: Fix bugs, Add new feature  
   A: Fix bugs (Quick Win)  
   ---  
   Q: Prioritize: [Your List]  
   A:  
   ```  

3. **Instructional Context:**  
   Set role, format, and constraints.  
   *Prompt:*  
   `"Act as a business architect. Output in table format. List pros/cons of cloud migration."`  

#### **B. Advanced Reasoning Frameworks**  
1. **Chain-of-Thought (CoT):**  
   Force step-by-step reasoning.  
   *Prompt:* `"Solve step-by-step: If 3x + 2 = 14, what is x?"`  

2. **Tree of Thoughts (ToT):**  
   Explore multiple reasoning branches.  
   *Use Case:* Complex scenario planning.  

3. **Graph of Thoughts (GoT):**  
   Interconnect ideas in a graph. Ideal for systems thinking.  

4. **Skeleton-of-Thought (SoT):**  
   Provide a partial structure for AI to complete.  
   *Prompt:*  
   ```  
   Complete this skeleton:  
   Problem: [P]  
   Root Cause: ______  
   Solution: ______  
   ```  

5. **Chain-of-Verification (CoVe):**  
   Self-correcting outputs.  
   *Prompt:*  
   `"Explain quantum computing. Then verify accuracy of each statement."`  

#### **C. Optimization & Automation**  
1. **Automatic Prompt Engineering (APE):**  
   AI optimizes prompts automatically.  
   *Tool:* **PromptPerfect**, **LangChain**.  

2. **Retrieval-Augmented Generation (RAG):**  
   Ground responses in external data (databases, docs).  
   *Use Case:* "Answer questions using our internal handbook."  

3. **Nexus (OpenRouter Search):**  
   Unified API to query multiple AI models simultaneously.

## 1. 7-Step Problem-Solving Process

This is the strategic backbone—how a human/AI “architect” should approach complex problems from definition to implementation.

### Step 1: Define the Problem – *Think Impact*
- **Basic Question**  
  - What is the core problem or decision?  
  - Clarify scope, time horizon, and success metrics.
- **Constraints & Stakeholders**  
  - Who is affected?  
  - What are hard constraints (regulatory, technical, financial, ethical)?  
  - What assumptions are being made?

---

### Step 2: Structure the Problem
- **Issue Trees**  
  - Break the main problem into sub-issues.  
  - Ensure logical decomposition (e.g., “Why is X happening?” → branches).
- **Hypothesis Trees**  
  - Translate issues into testable hypotheses (“X is due to Y because Z”).  
  - Align early thinking around what might be true.
- **MECE Principle (Mutually Exclusive, Collectively Exhaustive)**  
  - Ensure branches are non-overlapping and cover all relevant possibilities.

---

### Step 3: Prioritise Issues
- **High/Low Impact**  
  - Rank issues by potential effect on the outcome or key metrics.
- **Ease of Implementation**  
  - Classify by difficulty: technical, organizational, financial.
- **Quick Wins**  
  - Identify actions with high impact and low effort for early traction.

---

### Step 4: Issue Analysis & Work Plan
- **Hypotheses**  
  - Explicitly list what you aim to confirm or refute.  
- **Supporting Rationale**  
  - Why each hypothesis might be true (theory, precedent, domain knowledge).  
- **Data Sources**  
  - Internal data, external benchmarks, expert interviews, literature, logs, etc.  
- **Fact-Based Research**  
  - Plan specific analyses, experiments, or data pulls for each hypothesis.  
- **Model Exploration**  
  - Identify analytical or AI models to use (statistical, ML, simulation, LLMs).

---

### Step 5: Analysis
- Execute the work plan:
  - Clean and explore data  
  - Run models  
  - Test hypotheses  
  - Perform scenario and sensitivity analysis  
  - Assess limitations and uncertainties  

---

### Step 6: Synthesis
- **Pyramid Principle**  
  - Start with the main conclusion, then key supporting points, then details.  
- **Governing Thoughts**  
  - 1–3 overarching insights that organize all findings.  
- **Actionable Findings**  
  - Translate analysis into clear implications and decision options.

---

### Step 7: Recommendations
- **Client Commitment**  
  - Align with decision-makers on trade-offs and chosen path.  
- **Implementation Plan**  
  - Phasing, owners, timelines, KPIs, and change management.  
  - Risks, mitigations, and feedback loops.

## 2. Prompt Engineering Techniques

These are how you operationalize the 7-step process with AI—designing prompts and workflows so the model behaves like a disciplined problem-solver.

### Basic Prompting
- **Zero-Shot**  
  - Ask a direct question with no examples.  
- **Few-Shot**  
  - Include examples of desired input–output patterns.  
- **Instructional Context**  
  - Provide role, audience, goals, constraints, and format requirements.

---

### Reasoning Frameworks
- **Chain-of-Thought (CoT)**  
  - Ask the model to reason step-by-step.  
- **Tree of Thoughts (ToT)**  
  - Branch multiple reasoning paths; explore and compare options.  
- **Graph of Thoughts (GoT)**  
  - Interconnected reasoning paths; good for complex, interdependent issues.  
- **Logical Thinking (LoT)**  
  - Force explicit premises, inferences, and conclusions.  
- **Algorithm of Thoughts (AoT)**  
  - Impose algorithmic structures (search, dynamic programming, etc.).  
- **Skeleton-of-Thought (SoT)**  
  - First produce an outline, then flesh out each section.  
- **Rephrase and Respond (RaR)**  
  - Model restates the problem in its own words before answering.  
- **Chain-of-Verification (CoVe)**  
  - Separate answer generation and answer verification/checking.  
- **Step-Back Prompting**  
  - Ask the model to abstract: “What is this problem really about?” or  
    “Reframe this at a higher level, then solve.”

---

### Efficiency & Optimization
- **Automatic Prompt Engineering (APE)**  
  - Use the model (or tools) to search for and refine more effective prompts.  
- **Retrieval-Augmented Generation (RAG)**  
  - Ground responses in external knowledge sources (documents, DBs, APIs).  
- **Model Context Protocol (MCP)**  
  - Standardize how the model interacts with tools and external systems.  
- **Nexus (OpenRouter Search)**  
  - Use external search to augment reasoning with live or broader web data.

---

  - Issue-tree generators  
  - Hypothesis planners  
  - Auto-RAG for domain docs  
  - Verification and validation passes  
  - Report and slide generation
