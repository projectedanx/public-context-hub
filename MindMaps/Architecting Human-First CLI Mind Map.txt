Human-first design (Primary goal)
Serve both humans and programs (Central challenge)

Minimize user effort (Smart defaults, Auto-filled values)
Avoid ambiguity

Structure
<
Command Autocompletion (e.g., Tab to complete, Complete values)
Human-readable output is paramount
Machine-readable output (e.g., --json) for scripting
Proper use of STDOUT (API for other programs)
Standard Error (stderr) is for non-normal output (Warnings, Progress, Errors)
Always indicate progress/status (Keep user in loop)
Suggest next commands (Workflow guidance)
Fail gracefully (Informative and friendly errors)

Color and Formatting
<
0 = Successful execution
Non-zero = Failure (1=general failure)
Range: 0 ≤ exit code ≤ 123 (Shell uses 124-127, >128 for signal receipt)
Convey semantic meaning of error/status
Standardized constants (sysexits.h)
Rust/Golang: Compile static binaries for easy distribution (No dependencies)
Python: Requires dependency management (virtualenv, pip)
Node.js: Uses package manager (npm)
Containerize the CLI (Docker image) for dependency-free use
Rust: Better handling of breaking API changes (Compiler errors)
Use command-line parsing frameworks (e.g., Click, argparse, clap, cobra)
Avoid manually building filesystem paths (Use path.join)
Use double ampersand (&&) or double pipe (||) for command chaining (Cross-platform)
Implement custom actions/logic (e.g., argparse.Action, Click decorators)
