Definition: LLM crafts or refines its own prompt/instructions
Focus: Structural template (the 'How') over content (the 'What')
Distinction from CoT: Defines the steps, not just 'think step by step'
Architectural Layers
<
Chain-of-Thought (CoT): Explicit step-by-step reasoning
ReAct: Reasoning and Acting synergized
Reflection/Self-Refine: Iteratively refining output based on internal/external feedback
Refusal Breaker: LLM rewrites question if refused
Question Refinement: LLM suggests better version of prompt
Reinforced Prompt Optimization (RPO): Iterative prompt update based on natural language feedback (TD error)
Template Pattern: Ensures precise output format (e.g., URL, JSON, XML)
Add Clear Syntax: Using punctuation, headings, or markup (e.g., XML tags) to guide parsing
Specify Output Structure: Enforces consistency (e.g., tabular, tuple structure)
Evals: Systematic prompt test cases and metrics
Output Automater Pattern: Generates automation artifacts (scripts/code blocks)
Teach Attack Detection: Explicit instructions on common attack patterns
Use <thinking> and <answer> Tags: Separates internal reasoning from final output
Salted Sequence Tags: Wrap instructions in unique tags for isolation
Give the Model an 'Out' (Escape Hatches): Instructing model to admit uncertainty instead of guessing
Specifying Output Structure with Citations: Forces grounding (reduces fabrication risk)
RAG (Retrieval-Augmented Generation): Incorporates external, domain-specific context (prevents outdated/fabricated reliance)
Keep it Simple (K): One clear goal
Easy to Verify (E): Clear success criteria (e.g., 'include 3 code examples')
Reproducible Results (R): Avoid temporal references, use specific versions
Narrow Scope (N): One prompt = one goal (split complex tasks)
Explicit Constraints (E): Tell AI what NOT to do ('No external libraries')
Logical Structure (L): Context -> Task -> Constraints -> Format
