Configuration and Usage
<
Model Context Protocol (MCP)
<
Allows Linux environment in Windows without full VM overhead
WSL 1: Compatibility layer (implements Linux system calls)
WSL 2: Real Linux kernel in lightweight Hyper-V VM (full syscall compatibility, improved I/O)
wsl.conf: Per-distro settings (location: /etc/wsl.conf)
<
.wslconfig: Global settings (location: %UserProfile%\.wslconfig)
<
Key Criticism: Slow disk I/O performance
Symptom: npm/Yarn operations extremely slow when files are in Windows mounted folders (/mnt/c)
Workaround/Fix: Move project files to Linux home directory (~ or /home/username/)
Fixing Issues: wsl --shutdown (fast restart for WSL 2 config changes)
Known Issue: Gemini CLI access failure to /mnt/c via symlinks
