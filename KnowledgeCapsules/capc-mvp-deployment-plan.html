Cloud-Agnostic Personal Compute (CAPC) MVP Deployment Plan

The CAPC system is designed as a socio-technical intervention for digital inclusion. It repurposes aging, underutilized hardware into secure 'Digital Sovereignty Shells' that interface with low-cost cloud compute nodes, ensuring high-performance computing is accessible to low-income populations without the need for new hardware purchases.

Core Objective

Equitable Access via Legacy Repurposing

To deliver a technically functional, economically sustainable, and cognitively accessible system that fosters user autonomy and data integrity through open-source tools and transparent cost governance.

  • Affordability
  • Sustainability
  • Empowerment

Foundational Principles

The architecture balances radical minimalism with strategic software selection to reduce cognitive load and resource consumption.

The Digital Sovereignty Shell

The local client is repurposed using Linux Lite, a gateway OS designed for Windows users. It acts as a secure, minimalist environment that abstracts complex networking through custom scripts while maintaining local control over sensitive data.

Epistemic Green Budgeting

A user-centric financial framework where budgets are reframed as knowledge tools. Users set a 'green zone' for spending, supported by automated alerts and circuit breakers to prevent bill shock and promote agency.

Semantic Risk Cartography

A proactive UX design method that maps gaps between technical reality (e.g., latency) and user mental models. It uses the 'Intent Delta Dashboard' to visualize connection quality and costs in plain language.

Lightweight VC Framework

A security model using a simplified Verifiable Credential approach. The system signs SSH public keys during onboarding, ensuring the Cloud VM only accepts connections from 'blessed' authenticated users.

MVP Deployment Lifecycle

A phased journey designed to reach the 'Aha!' moment of cloud compute access with minimal technical friction.

Step 1

Web Portal Onboarding

User signs up, selects an Epistemic Green Budget, and triggers backend automation for GCP project provisioning.

  • •Budget slider selection
  • •Geographic region optimization
  • •Automated SSH key generation
Step 2

Local Shell Configuration

The user converts legacy hardware using a pre-packaged ISO and simplified installer.

  • •Linux Lite installation
  • •Cryptomator vault creation
  • •CAPC-setup script deployment
Step 3

Connection & Validation

The user launches their Cloud PC through a single-icon interface, triggering a secure VNC-over-SSH tunnel.

  • •SSH tunnel establishment
  • •Mutual authentication check
  • •VNC session initialization

Risk Mitigation & Reflexivity

Addressing semantic gaps and potential unintended harms through reflexive critique.

Latency/Network Instability

High network delay makes the remote desktop feel unresponsive.

Mitigation Strategy

  • →Real-time 'Connection Quality' widget
  • →Regional data center optimization
  • →XFCE desktop environment tuning

Cost Overrun / Bill Shock

Unexpected egress costs lead to financial stress.

Mitigation Strategy

  • →Programmatic shutdowns via Pub/Sub
  • →Positive friction modal at 90% budget
  • →Intent Delta Dashboard visualization

Algorithmic Trauma

Budget enforcement feels punitive rather than protective.

Mitigation Strategy

  • →Informational nudges before hard stops
  • →Transparent 'Epistemic' framing
  • →Human-in-the-loop support escalation

Pre-Deployment Readiness

Essential steps to ensure the security and sustainability of the pilot environment.

Hardware & OS Verification

  • Legacy PC meets 1.0 GHz / 1GB RAM minimum
  • Linux Lite 7.x installed and configured
  • TigerVNC viewer verified

Cloud Infrastructure Setup

  • GCP Free Tier status confirmed
  • e2-micro instance template created
  • AIDE HIDS initialized in read-only storage

Economic Controls

  • Pub/Sub topics for budget triggers active
  • Cloud Functions for programmatic shutdown tested
  • Instance schedules set for daily idle-off
