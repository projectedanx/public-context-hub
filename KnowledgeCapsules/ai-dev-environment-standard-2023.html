Corporate AI Development Environment Policy

This policy establishes a unified architectural foundation for AI development, targeting the elimination of configuration drift and the maximization of hardware performance. By mandating specific filesystem architectures and containerized workflows, the organization ensures that every engineer operates within a secure, high-throughput environment optimized for Large Language Model (LLM) and Machine Learning (ML) development.

Architectural Mandate

Performance-First Infrastructure

The policy centers on three pillars: WSL2 for native Linux performance, VSCode as the primary AI-integrated control plane, and Dev Containers for immutable environment reproducibility across distributed teams.

  • Security
  • Performance
  • Reproducibility

Core Architectural Pillars

The following concepts represent the non-negotiable foundations of the corporate engineering standard.

Filesystem Locality

To avoid the significant I/O performance penalties of cross-OS communication, all project files, datasets, and virtual environments must reside strictly within the native Linux filesystem. This ensures 'Git' and 'npm' operations remain at near-native speeds.

Declarative Environments

Standardization is achieved through 'Dev Containers' (devcontainer.json). These artifacts define the OS, system-level dependencies, and IDE extensions in code, ensuring the 'works on my machine' problem is eliminated through containerization.

AI-Augmented Iteration

Development is redefined as a multi-step dialogue between the engineer and a managed AI assistant (GitHub Copilot). This workflow prioritizes specification and validation over manual syntax entry, utilizing a 'Prompt-Generate-Refine' cycle.

The Standard AI Development Cycle

A codified approach to leveraging AI tools consistently across the engineering organization.

Step 1

Prompt & Specification

Begin feature development by defining high-level logic in comments or the chat view to provide the AI with context before any code is written.

  • •Define inputs/outputs clearly
  • •Reference architectural patterns in .github/copilot-instructions.md
Step 2

Validation & Debugging

Execute generated scaffolding immediately within the integrated debugger to verify correctness and catch hallucinated logic early.

  • •Use VSCode integrated debugger
  • •Engage 'Fix with Copilot' for rapid error resolution
Step 3

Quality Assurance & Documentation

Finalize the feature by using the AI to generate unit tests and descriptive, conventional commit messages for version control.

  • •Generate pytest or unittest suites
  • •Auto-generate diff-based commit messages

Compliance & Setup Checklist

Actionable steps to ensure your local environment adheres to corporate policy.

Initial Environment Setup

  • Verify WSL2 is in 'mirrored' networking mode
  • Ensure projects are stored in ~/projects/ on Linux
  • Run install_extensions.sh for mandated VSCode tools

Security & Governance

  • Verify no secrets are hardcoded in source code
  • Confirm GitHub Copilot Business plan is active
  • Ensure .github/copilot-instructions.md exists in repository

Maintenance

  • Perform VHDX compaction monthly to reclaim disk space
  • Run 'docker system prune' regularly inside WSL
  • Audit running extensions for performance bottlenecks
