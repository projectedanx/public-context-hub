Collaborative Evolution of Affordance Ontologies

This architectural framework enables autonomous AI agents within plugin-rich environments to move beyond simple tool usage into active 'meaning making.' By formalizing software components as computable affordances, agents can negotiate and reach consensus on the function of tools in a dynamic, ever-changing ecosystem.

Framework Core

Distributed Semantic Governance

A system for managing the evolution of software meaning through agentic observation, structured argumentation, and immutable logging of ontological consensus.

  • Meaning Makers
  • Autonomous Agents
  • System Integrity

Core Theoretical Pillars

The system is built on the intersection of ecological psychology and multi-agent system theory.

Computational Software Affordance

Defined as a tuple ⟨T,C,E,P⟩ representing the Tool, required Capabilities, probable Effects, and necessary Properties. This transforms software utility from a static property into a computable, relational model.

Delegative Semantic Democracy

A consensus model where agents utilize Liquid Democracy to delegate voting power on specific technical domains (e.g., security) to 'guru' agents, ensuring expertise-driven governance.

Dual-Graph Ontological Memory

Separates transient Local Beliefs (speculative agent observations) from the Shared Consensus Graph (validated source of truth), allowing for structured negotiation without corrupting the core knowledge base.

The Lifecycle of Meaning

Meaning is constructed through a cyclical process of observation and collective validation.

Step 1

Change Discovery

Agents monitor the environment for plugin updates, code-level differences, or behavioral anomalies in sensory hook streams.

  • •Hook-based event monitoring
  • •Code-level diff analysis
  • •Version change tracking
Step 2

Semantic Negotiation

The Agentic Affordance Proposal Protocol (AAPP) is triggered, initiating a debate using formal grammars to propose, challenge, or accept new meanings.

  • •AAPP document submission
  • •FIPA ACL argumentation
  • •Trust-weighted voting
Step 3

Consensus Finalization

Once a supermajority threshold is reached, the new affordance is cryptographically signed and recorded on the ledger.

  • •Quorum verification
  • •Reputation staking resolution
  • •DASL record appending

System Resilience

Mechanisms to protect the system from corruption, drift, and structural failure.

Adversarial Voting Blocs

Malicious agents forming collusive groups to force incorrect meanings into the ontology.

Mitigation Strategy

  • →Reputation Staking
  • →Witness-based interaction trust
  • →Certified Credentials

Semantic Cold Start

New plugins or agents lacking sufficient history to generate high-trust evidence.

Mitigation Strategy

  • →Affordance Priors via Deep Learning
  • →Bootstrapping from trusted peer ledgers
