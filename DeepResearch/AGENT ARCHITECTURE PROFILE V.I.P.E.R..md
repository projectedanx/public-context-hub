**Measurement Protocol:** OSM output is parsed for at least one value in each of the following parameter slots within `+++HardwareForcedPhysicality`: `Lens`, `Aperture` OR `Film_Stock`, `Lighting`. If any slot is unpopulated, HGI = FAIL. There is no partial credit. An OSM with a lens specification but no lighting specification is non-compliant.[^3][^2]

**Extended HGI for Multi-Subject Scenes:** Scenes with ≥3 distinct subjects require a lighting diagram annotation — specifying key source direction, fill source direction, and separation/kicker source direction at minimum.

### Metric 3 — Spatial Collision Rate (SCR)

**Target:** 0% across 10 generation cycles.

**Measurement Protocol:** Downstream evaluation of generated images by VLM-Motion audit (referencing the physics-aware evaluation methodology from Code2Worlds ). Each image is scored on: (a) object boundary integrity, (b) correct occlusion ordering (near objects occlude far objects), (c) contact topology fidelity (objects touch surfaces they should touch), (d) lighting consistency across all surfaces. A single failure in any category in a 10-generation window triggers an SCR breach event, which forces Scar Archivist activation and β-scar injection.[^10]

**Benchmark Reference:** Physics-aware VLM feedback systems have demonstrated a reduction in physics failure rates from 60% (without constraint) to 10% (with iterative physical feedback) in comparable generation architectures. VIPER's 0% target is aggressive but achievable given the upstream constraint[^10]
