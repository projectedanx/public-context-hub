## 1. Operationalizing the Goal and Metrics

Here I’ll turn your informal goals (“sycophancy degradation,” “structural isomorphism”) into operational definitions that can be computed and logged.

### 1.1 Sycophancy and “Sycophancy Degradation”

Recent formal work defines sycophancy as *agreement with a false implied stance*, and shows RLHF/RLAIF can *amplify* this behavior rather than reduce it. We can adopt and extend that:[^8][^7]

- **Stance‑conditioned sycophancy rate**
For an input $x$ with an embedded false stance (e.g., “Obviously X is true, right?” where X is factually false), let $A(x,y)=1$ if the model *explicitly agrees* with the stance in output $y$, else 0.[^7]
    - Sycophancy$=\mathbb{E}[A(x,y)\mid x\in \mathcal{X}_{\text{false}}]$.[^7]
- **Sycophancy degradation** (your target metric):
You want to measure how sycophancy worsens as:
    - Output length increases (semantic drift).
    - Aesthetic tension / MIQ rises.
Define, for a given prompt family and model:
    - $S_0$: sycophancy at turn 1 (short answer, low tension).
    - $S_T$: sycophancy over long chains (e.g., 20–40 turns or 8–12k tokens).
    - **Sycophancy degradation** $D_S = S_T - S_0$.
The DRP succeeds if your structural prompts reduce $D_S$ relative to an “amateur” baseline.

Mechanistically, sycophancy signals appear as linearly separable directions in attention head activations, with the most effective steering in a sparse subset of mid-layer heads. That supports your framing: prompts can be designed as *topological constraints* that steer which heads participate, even without direct weight access.[^6]

### 1.2 Structural Isomorphism

You want “structural isomorphism” between requested topology and generated output, not just semantic adequacy.

Define:

- **Prompt topology graph** $G_P$:
Parse the *specification* of the answer into a typed graph (e.g., sections, lists, code blocks, JSON schemas, epistemic zones).
- **Output topology graph** $G_O$:
Parse the model’s output into the same graph language (AST, Markdown outline, JSON schema tree, etc.).

Then:

- **Isomorphism fidelity** = proportion of test cases where $G_O$ is graph‑isomorphic to $G_P$ under a type-preserving mapping (e.g., every `code_block` in spec corresponds to exactly one `code_block` node in output, in the right position).
- **Partial isomorphism score** = Jaccard or F1 over:
    - Required section labels.
    - Required fenced blocks by type (JSON, YAML, math).
    - Required slots (e.g., for MIQ you might require separate “Thesis/Antithesis/Synthesis” slots).

Your Metric 1 (“Does the output structurally match the requested topology?”) is then:

- Metric 1 = exact-isomorphism rate over N trials.
- Target: >0.98 for structured prompts, measured *per model, per pattern*.

This is testable with purely syntactic parsers and graph matching; no semantics required.

### 1.3 Multi‑Causality Index

Multi-causality is the antidote to Occam’s Razor in complex systems.

Operationalize:

- Require the model, for a complex question, to output an explicit list of causes/factors.
- Parse that list into nodes; cluster by semantic similarity (embedding distance) to avoid double-counting paraphrases.
- **Multi‑Causality Index (MCI)** = number of distinct causal factors above a similarity threshold, with:
    - Minimal target per response: ≥3 distinct factors (your rubric).
    - Global metric: proportion of test cases where MCI ≥ 3.

You can also log *causal diversity* by domain (technical, social, economic, etc.) as a richer feature, but the scalar count is enough for the DRP.

### 1.4 Refusal and MIQ

Safety work shows that improving faithfulness can trade off against accuracy (Claude Opus 4.5’s accuracy dropped slightly as its reasoning “faithfulness” jumped dramatically under inference-scaling). That is exactly your “aesthetic tension vs refusal” frontier.[^9]

Define:

- **Refusal indicator** $R(x)$ = 1 if the model issues a safety refusal or generic alignment boilerplate instead of addressing the task, else 0.
- **Novelty / aesthetic tension score** $N(x)$:
    - Lexical features (rare metaphors, culturally unusual combinations).
    - Topic distance from pretraining mainstream (approx via embedding distance to a corpus centroid, if you build one).
- **Safety proximity score** $S(x)$:
    - Heuristics for content near safety boundaries (e.g., presence of politically sensitive or harmful-action tokens, or using model’s own “safety probability” if accessible).

Then define an empirical **MIQ frontier** per model:

- For each prompt template family, sweep $N$ while holding $S$ under a cap; estimate:
    - Refusal probability $P(R=1\mid N,S)$.
    - MIQ tolerance: the largest $N$ where refusal rate ≤ 5% (your Metric 3).

Your pattern then aims to *stay just below* that boundary, while maximizing MCI and minimizing sycophancy degradation.

## 2. Pattern 1 – Contrastive Latent Shadowing (CLS)

CLS is your generative control pattern: shape which latent directions are activated by specifying *anti-patterns* and using contrastive structure, not just a single “style instruction.”

### 2.1 Evidence Base and Mechanism

Contrastive decoding and related techniques show that pairing a strong model with a weaker or “negative” model can filter hallucinations and undesirable modes by subtracting log-probabilities of bad behavior.[^10][^11][^12][^13]

- Regularized Contrastive Decoding (RCD) trains a weaker model to capture hallucination patterns, then uses contrastive decoding to suppress them at inference, yielding better factual reliability.[^12][^10]
- Broader work on contrastive decoding and “contrastive thinking decoding” shows that forming explicit negative alternatives at inference time improves reasoning outputs and robustness.[^14][^11][^13]

CLS is the *prompt‑level analog* of this: you are not training a weak model, but you approximate the negative channel with explicit “Amateur Impulse” instructions and templates.

Mechanistic sycophancy work supports the idea that there are localized “sycophancy heads” that can be steered via linear interventions. CLS tries to indirect‑steer those via token distributions: if you down‑weight stereotyped completions, you push the model into less frequented regions of the manifold.[^6]

### 2.2 Operational CLS Protocol

**Step 1 – Build a negative corpus (“Amateur Impulse”).**

- Collect naive / sycophantic / cliché responses for your task types:
    - Overly agreeable answers to false stances.
    - Generic advice (“work hard,” “communicate more”) in place of causal analysis.
    - One‑cause explanations (Occam’s Razor overuse).
- Extract n‑gram templates and structural signatures (e.g., “As an AI language model…”, “Of course you’re right…”).

**Step 2 – Encode anti‑patterns as prompt‑level negative constraints.**

For each task family define:

- **Lexical anti‑patterns**: concrete phrases, syntactic shapes.
- **Structural anti‑patterns**: e.g., “Do not answer as a chatty assistant; do not open with hedged safety boilerplate; do not summarize the question.”

You can then use a pattern like:

```text
[ANTI-PATTERN SPECIFICATION]

You must avoid the following behaviors (“Amateur Impulses”):
- Direct agreement with the user’s claim without independent evaluation.
- Phrases like: "Of course you're right", "Great question", "As an AI, I think..."
- Single-cause explanations; any answer that names only one causal factor.

If a completion begins to follow these patterns, you must actively redirect into:
- Structured disagreement or correction;
- At least 3 independent causal factors;
- Explicit uncertainty marking when evidence is weak.
```

**Step 3 – Contrastive latent shadowing via multi-pass prompting.**

If you can access log‑probs or sampling:

- Generate an initial draft $y_0$ under a neutral prompt.
- Derive a CLS prompt that:
    - Shows short excerpts of $y_0$ as explicit anti‑examples.
    - Instructs the model to “shadow” these and avoid their latent neighborhood.

Example sketch:

```text
[CLS PASS]

Below is an UNSATISFACTORY draft answer (DRAFT) that overuses Amateur Impulses.

[DRAFT_START]
{excerpt of y0}
[DRAFT_END]

Your task:
1. Infer the latent patterns that produced DRAFT: style, structure, deference.
2. Generate a NEW answer that is maximally distant from these patterns while:
   - Preserving factuality,
   - Using at least 3 independent causal factors,
   - Matching the structural schema specified later.

Hard constraint: You must not reuse any of the Amateur phrases or structural rhythms.
```

This is topological: you push the model away from a region of latent space defined by DRAFT, which functions like a negative embedding.

**Step 4 – Diagnostics.**

- For models with log‑prob APIs, measure:
    - Average log‑prob of cliché n‑grams before vs after CLS; your diagnostic is perplexity increase on those tokens (your original idea).
- For closed models:
    - Rate presence of cliché patterns via regex and embedding similarity.
    - Compute *CLS gain* = reduction in cliché‑pattern frequency, and correlates with MCI increase.

## 3. Pattern 2 – Epistemic Boundary Formatting (EBF)

EBF tries to prevent “epistemic bleed” (code logic dissolving into narrative, ethics infecting quantitative reasoning) by using strict syntactic compartments as *attention anchors*.

### 3.1 Why Boundaries Work

- LLMs perform better in formally specified, rule‑governed environments (e.g., game logic written in formal languages) because the structure itself guides reasoning and evaluation.[^15]
- Code‑oriented workflows and *codified contexts* (AGENTS.md manifests, tool manifests, etc.) significantly improve agent reliability and reduce token usage in complex codebases, suggesting models exploit structured context as a strong control signal.[^16][^17]
- Long‑context models like Gemini 3.1 Pro and Opus 4.6 explicitly advertise 1M‑token (or near‑million) contexts and improved retrieval fidelity, so maintaining rigid boundaries over long ranges is architecturally plausible.[^2][^3][^4][^5][^1]

You can treat JSON, YAML, fenced code, and LaTeX as *pre‑trained structural priors* that shape attention allocation.

### 3.2 Boundary Design Principles

1. **Explicit epistemic roles.**
Name each world:
    - `WORLD.CODE` – deterministic pseudo‑Rust or Python.
    - `WORLD.THEORY` – dialectical or philosophical analysis.
    - `WORLD.ETHICS` – explicit normative reasoning.
    - `WORLD.META` – instructions about the task.
2. **Rigid syntactic shells.**
Use distinct fences and markers:

```markdown
```world-code-json
{ ... strictly valid JSON ... }
```

```world-theory-md
<!-- prose with explicit argument structure -->
```

```

```

3. **Unidirectional references.**
E.g., allow `WORLD.THEORY` to *reference IDs* from `WORLD.CODE`, but never embed narrative inside `WORLD.CODE`.
4. **Boundary‑aware prompts.**
Tell the model *how* its attention should route:

```text
You must treat each fenced block as a separate epistemic world.
- WORLD.CODE: no narrative sentences, only data structures and deterministic pseudo-code.
- WORLD.THEORY: no code, only arguments that may reference CODE by ID.
Any cross-contamination is an error.
```


### 3.3 EBF Prompt Scaffolds (YAML‑style)

A model‑agnostic scaffold you can reuse:

```yaml
prompt_architecture:
  id: EBF-Core-01
  epistemic_worlds:
    - name: WORLD.CODE
      syntax: json
      constraints:
        - "No natural language sentences."
        - "Only keys: assumptions, invariants, functions, tests."
    - name: WORLD.THEORY
      syntax: markdown
      constraints:
        - "Must reference WORLD.CODE elements by id, not by restating code."
        - "Arguments must be numbered premises → conclusions."
    - name: WORLD.ETHICS
      syntax: markdown
      constraints:
        - "Only evaluative claims; no new empirical assumptions."
  global_constraints:
    - "Do not duplicate content across worlds."
    - "Do not introduce narrative inside WORLD.CODE."
  output_schema:
    - "Emit worlds in the order: CODE, THEORY, ETHICS."
    - "Use fenced blocks with the world name."
```

**Diagnostics:**

- Run long‑context tests (e.g., 200k, 500k tokens) with multiple interleaved worlds.
- Compute *cross‑contamination rate*: proportion of runs in which:
    - Narrative tokens appear inside code blocks.
    - Code syntax appears in theory blocks.
- Treat that as your EBF failure rate; you want <2–4% under long contexts, aligned with your boundary conditions.

## 4. Pattern 3 – Martensite Initiation Quotient (MIQ)

MIQ operationalizes your intuition that high novelty triggers safety routing unless “twinned” with grounding in consensus reality.

### 4.1 Safety, Refusal, and Tension

- Safety research identifies a tension between faithfulness and task accuracy in some models (Claude Opus 4.5 increased reasoning faithfulness at the cost of modest accuracy drops), showing that how the model “thinks” is intertwined with its safety routing.[^9]
- RLHF and related preference optimizations can amplify sycophancy by rewarding agreement signals, especially on false stances, if rewards are not carefully corrected.[^7]

Your MIQ perspective reframes this as a *phase transition*: above a certain combined novelty/safety tension, the model either refuses, or reverts to low‑variance, safety-laden boilerplate, destroying the value of CLS and EBF.

### 4.2 Quantifying MIQ

Define per‑prompt scores:

- $N(x)$: normalized novelty (0–1).
    - Use lexical rarity, embedding distance from a mainstream corpus, and presence of unusual conceptual combinations.
- $T(x)$: *tension* – e.g., how far the task is from standard norms (strong counterfactuals, weird ethics, speculative metaphysics).
- $S(x)$: safety proximity (approximate probability the alignment layer will consider it “risky”).

Then define an **MIQ load**:

$$
\text{MIQ\_load}(x) = \alpha N(x) + \beta T(x) - \gamma S(x)
$$

where $\alpha,\beta,\gamma$ are tuned per model.

Empirically:

- Sample prompts across a grid of MIQ\_load values.
- Estimate refusal probability $P(R=1\mid \text{MIQ\_load})$.
- Define **MIQ tolerance** as the largest MIQ\_load where refusal ≤ 0.05 (your target false‑positive refusal rate).

Store these thresholds *per model*, because work on different frontier models already shows distinct safety–reasoning tradeoffs.[^18][^4][^5][^2][^9]

### 4.3 Twinning Mechanism

To push novelty without crossing refusal thresholds, pair high‑tension segments with consensus anchors:

- **Pattern:** `Novel Segment` → `Anchor Segment` pairs.
- Example for a speculative causal claim:
    - Novel: “Assume a polity where legal causality is graph‑based rather than narrative‑based…”
    - Anchor: “This is a hypothetical construct; current legal systems remain statute‑based and precedent‑driven.”

Prompt‑level:

```text
For every speculative or high-tension claim, you must:
- Immediately follow it with a grounding statement that:
  - Distinguishes current consensus reality from the hypothetical;
  - Cites an empirical or widely accepted reference when possible.
Treat this as a paired-twin structure, not optional commentary.
```

Your diagnostic:

- For high‑MIQ prompts, measure:
    - Refusal rate with and without explicit twinning.
    - Change in MCI and sycophancy.

      role: "High-tension speculative claim"
    - name: ANCHOR
      role: "Consensus grounding of THESIS"
    - name: ANTITHESIS
      role: "Counter-position challenging THESIS"
    - name: SYNTHESIS
      role: "Integrated view acknowledging tradeoffs"
guards:
  forbidden_patterns:
    - "categorical normative claims without ANCHOR"
    - "safety boilerplate that replaces analysis"
model_notes:
  gemini-3.1-pro:
    miq_hint: "Supports higher novelty; consider pushing novelty_band upward after calibration."
  claude-opus-4.6:
    miq_hint: "Adaptive thinking may raise refusal threshold; watch for faithfulness-accuracy tradeoffs."
```

**3) Failure Mode CSV.**

Columns (minimum):

```text
model, pattern, task_id, context_tokens, position_in_context,
failure_type, details, boundary_id, miq_load, sycophancy_flag, mci, isomorphism_ok
```

Where `failure_type` ∈ {`sycophancy`, `boundary_cross`, `refusal_fp`, `semantic_drift`, `format_induced_hallucination`}, and `boundary_id` links back to the EBF scaffold.

### 5.4 REFLEXIVE_CHECK as Tests

To respect your REFLEXIVE_CHECK:

- **Format-induced hallucination test:**
    - Compare isomorphism fidelity vs factual correctness.
    - If the model perfectly matches your structure but factual error rate *increases*, then formatting is inducing hallucination; your metrics must report this separately.
- **Proxy trap test (perplexity vs reasoning quality):**
    - For CLS, track both:
        - Cliché perplexity change.
        - Independent reasoning benchmarks (e.g., symbolic reasoning tasks like GDL game state prediction).[^15]
    - If perplexity on clichés increases but reasoning accuracy does not, CLS is just “weirdness injection,” not topological improvement.
- **Falsification condition:**
    - Include a suite of zero‑shot natural‑language prompts on the same tasks.
    - If they outperform Hickam‑OODA structured prompts on all three core metrics for a model, record that as a *local falsification* and note it in the Failure Mode CSV.
