## 11. Validation — Polyglot Hallucination Resonance Prevention

The final validation gate addresses the most systemic failure mode in multimodal generation pipelines: **Polyglot Hallucination Resonance (PHR)**. The two outputs are only composed at the HTML-to-PNG rendering stage — a purely mechanical spatial overlay operation where no semantic inference occurs.[^2]

The **AST parse validation** that precedes every ACT phase submission confirms structural validity of the JSON. This is the `+++DCCDSchemaGuard` final-pass check against the `Carousel_API_Manifest` DFA. Any JSONDecodeError, missing required field, out-of-bounds value (e.g., `slide_index: 21` for an Instagram target), or enum violation (e.g., `aspect_ratio: "16:9"`) is caught here and triggers `+++SagaRecovery` rollback before a single API call is made.[^14][^15]

The falsification test is simple and unambiguous: if the output of `carousel_payload_v1.json` fails `jsonschema.validate(payload, carousel_api_manifest_schema)` in standard Python, or fails to import cleanly into a Make.com JSON parse module or Hootsuite bulk upload, the architecture is declared **FLAWED**. The empirical evidence from arXiv:2602.12247 confirms that constrained decoding frameworks guarantee syntactically valid, schema-conforming output when the schema is correctly compiled into a grammar artifact — which is precisely what `+++DCCDSchemaGuard` implements.[^16][^14]
