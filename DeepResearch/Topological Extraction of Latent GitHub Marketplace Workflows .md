### Architecture 1: The Distributed Asynchronous State Machine (DASM)

**Pattern:** The Asynchronous State-Passer
**Lens Applied:** Resource Recombination / Exaptation
**Claim:** A matrix strategy, combined with artifact upload/download as a message bus, can implement a distributed finite state machine across parallel runner nodes with verified state coherence.

This architecture exploits the fact that `actions/upload-artifact` with a deterministic name and `actions/download-artifact` in a subsequent `needs:` job creates a *causally ordered message-passing channel* — exactly the primitive required by a distributed FSM. The matrix dimension provides parallelism; the artifact namespace provides the shared memory bus.[^2]

The critical non-obvious insight is **artifact namespace collision as intentional synchronization**. When five parallel matrix jobs all attempt to write `state-shard-N`, job-level uniqueness via `${{ matrix.shard }}` ensures partition-safe writes, and a downstream aggregation job reassembles the state by downloading all shards and computing a Merkle-like hash of their combined output — providing state coherence verification.

**Boundary Conditions:**
            - Artifact retention limit: 90 days default (configurable to 400 days)[^6]
            - Max artifact size: 10 GB per artifact, 100 GB total per workflow run
            - State coherence breaks if `fail-fast: true` causes partial matrix execution; always use `fail-fast: false`

```yaml

# Architecture: Distributed Asynchronous State Machine
# Topology Class: State Writer × Matrix Parallelism → Temporal Convergence
# Schema: GitHub Actions Workflow v2026

name: "DASM — Distributed Async State Machine"

on:
  workflow_dispatch:
    inputs:
      shard_count:
        description: "Number of parallel FSM nodes (2–16)"
        required: true
        default: "4"
        type: string
      state_input:
        description: "Base64-encoded initial state JSON"
        required: true
        type: string

env:
  STATE_VERSION: "v1"
  COHERENCE_THRESHOLD: "0.95"

jobs:
  # PHASE 0: State Initialization — Write initial state as artifact
  init-state:
    runs-on: ubuntu-24.04
    outputs:
      state_hash: ${{ steps.hash.outputs.hash }}
      matrix_config: ${{ steps.gen-matrix.outputs.matrix }}
    steps:
      - name: Decode and validate initial state
        id: decode
        run: |
          echo "${{ inputs.state_input }}" | base64 -d > initial_state.json
          python3 -c "import json,sys; json.load(open('initial_state.json')); print('State JSON valid')"

      - name: Generate dynamic shard matrix
        id: gen-matrix
        run: |
          SHARDS=$(python3 -c "
          import json
          n = int('${{ inputs.shard_count }}')
          matrix = {'shard': list(range(n)), 'include': [{'shard': i, 'role': ['compute','validate','transform','aggregate'][i % 4]} for i in range(n)]}
          print(json.dumps(matrix))
          ")
          echo "matrix=$SHARDS" >> $GITHUB_OUTPUT

      - name: Hash initial state
        id: hash
        run: |
          HASH=$(sha256sum initial_state.json | cut -d' ' -f1)
          echo "hash=$HASH" >> $GITHUB_OUTPUT
          echo "STATE_HASH=$HASH" >> initial_state.json

      - name: Upload initial state as message bus artifact
        uses: actions/upload-artifact@v4
        with:
          name: "fsm-state-init-${{ env.STATE_VERSION }}"
          path: initial_state.json
          retention-days: 1

  # PHASE 1: Parallel FSM Nodes — Each shard processes a partition of state
  fsm-node:
    needs: [init-state]
    runs-on: ubuntu-24.04
    strategy:
      fail-fast: false
      matrix: ${{ fromJSON(needs.init-state.outputs.matrix_config) }}
    outputs:
      shard_result_hash: ${{ steps.process.outputs.result_hash }}
    steps:
      - name: Download initial state from message bus
        uses: actions/download-artifact@v4
        with:
          name: "fsm-state-init-${{ env.STATE_VERSION }}"
          path: ./state-in

      - name: Verify state coherence (pre-processing)
        run: |
          EXPECTED="${{ needs.init-state.outputs.state_hash }}"
          ACTUAL=$(sha256sum ./state-in/initial_state.json | cut -d' ' -f1)
          # Extract stored hash from JSON
          STORED=$(python3 -c "import json; d=json.load(open('./state-in/initial_state.json')); print(d.get('STATE_HASH',''))")
          if [ "$ACTUAL" != "$STORED" ]; then
            echo "::error::State coherence violation on shard ${{ matrix.shard }}"
            exit 1
          fi
          echo "State coherence verified for shard ${{ matrix.shard }}"

      - name: Process state partition (role-aware transformation)
        id: process
        run: |
          python3 << 'EOF'
          import json, hashlib, os

          with open('./state-in/initial_state.json') as f:
              state = json.load(f)

          shard = int("${{ matrix.shard }}")
          role = "${{ matrix.role }}"

          # FSM transition function — role-partitioned computation
          result = {
              "shard_id": shard,
              "role": role,
              "input_keys": list(state.keys()),
              "transition": f"FSM_NODE_{shard}_PROCESSED",
              "timestamp": os.popen("date -u +%Y%m%dT%H%M%SZ").read().strip(),
              "role_output": {
                  "compute": {"cpu_score": shard * 42 + len(state)},
                  "validate": {"validity": True, "checks_passed": 7},
                  "transform": {"normalized_keys": sorted(state.keys())},
                  "aggregate": {"key_count": len(state), "shard_weight": 1.0 / int("${{ inputs.shard_count }}")}
              }.get(role, {})
          }

          with open('shard_result.json', 'w') as f:
              json.dump(result, f, indent=2)

          h = hashlib.sha256(json.dumps(result, sort_keys=True).encode()).hexdigest()
          with open(os.environ['GITHUB_OUTPUT'], 'a') as gh:
              gh.write(f"result_hash={h}\n")
          print(f"Shard {shard} ({role}) processed. Hash: {h}")
          EOF

      - name: Upload shard result to message bus
        uses: actions/upload-artifact@v4
        with:
          name: "fsm-shard-result-${{ matrix.shard }}-${{ env.STATE_VERSION }}"
          path: shard_result.json
          retention-days: 1

  # PHASE 2: State Aggregation — Merkle-verify all shard results
  aggregate-state:
    needs: [init-state, fsm-node]
    runs-on: ubuntu-24.04
    steps:
      - name: Download all shard results from message bus
        uses: actions/download-artifact@v4
        with:
          pattern: "fsm-shard-result-*-${{ env.STATE_VERSION }}"
          merge-multiple: true
          path: ./shard-results

      - name: Compute state coherence Merkle hash
        id: merkle
        run: |
          python3 << 'EOF'
          import json, hashlib, os, glob

          shards = []
          for f in sorted(glob.glob('./shard-results/*.json')):
              with open(f) as fp:
                  shards.append(json.load(fp))

          # Merkle tree over shard hashes
          leaf_hashes = [hashlib.sha256(json.dumps(s, sort_keys=True).encode()).hexdigest() for s in shards]
          merkle_root = hashlib.sha256(''.join(leaf_hashes).encode()).hexdigest()

          coherence_score = len(shards) / int("${{ inputs.shard_count }}")
          print(f"Shards received: {len(shards)}/{int('${{ inputs.shard_count }}')}")
          print(f"Merkle root: {merkle_root}")
          print(f"Coherence score: {coherence_score:.2f}")

          if coherence_score < float("${{ env.COHERENCE_THRESHOLD }}"):
              print(f"::error::Coherence {coherence_score:.2f} below threshold {os.environ['COHERENCE_THRESHOLD']}")
              exit(1)

          final_state = {"merkle_root": merkle_root, "shards": shards, "coherence": coherence_score}
          with open('final_state.json', 'w') as f:
              json.dump(final_state, f, indent=2)
          print("State machine convergence verified.")
          EOF

      - name: Upload final converged state
        uses: actions/upload-artifact@v4
        with:
          name: "fsm-final-state-${{ env.STATE_VERSION }}"
          path: final_state.json
          retention-days: 90

### Architecture 3: The Recursive Autopoietic Security Loop (RASL)

**Pattern:** Autonomous Self-Healing via IssueOps Closure
**Lens Applied:** Latent Need / Problem Space
**Claim:** By composing `github/codeql-action` (security scanner), `actions/github-script` (issue creation/parsing), and an LLM action (patch generation), a repository achieves autopoiesis — self-creation and maintenance of security patches with zero human intervention.

This architecture directly implements the **Recursive Autopoiesis Hypothesis** from Section 9 of the DRP protocol. The closed loop is: scan → detect vulnerability → create structured issue → parse issue via webhook → call LLM for patch → create PR → merge (or hold for human gate). The loop is topologically closed because the merged PR re-triggers the scan [][].

**Boundary Conditions:**
            - Requires `ANTHROPIC_API_KEY` or equivalent LLM API secret in repo secrets
            - `pull-requests: write` and `security-events: read` permissions required
            - Loop termination: Implement a `RASL_ITERATION_COUNT` counter in issue metadata to prevent infinite recursion (max depth: configurable, default: 3)

```yaml
# +++PetzoldSequence[DRP-GHA-RASL-003]
# Architecture: Recursive Autopoietic Security Loop
# Topology Class: State Reader (CodeQL) → Emission Node (Issue) → LLM Transform → Emission Node (PR)
# Autopoiesis: Closed-loop self-healing; requires human gate at iteration_count > threshold

name: "RASL — Recursive Autopoietic Security Loop"

on:
  push:
    branches: [main, develop]
  schedule:
    - cron: "0 3 * * *"  # Nightly security heartbeat
  issues:
    types: [opened, labeled]  # IssueOps re-entry point

permissions:
  contents: write
  security-events: read
  issues: write
  pull-requests: write

  RASL_MAX_DEPTH: "3"
  LLM_MODEL: "claude-opus-4-6"  # Q2 2026 Standard
  PATCH_BRANCH_PREFIX: "rasl/auto-patch"

  # GATE: Determine if this is a scan trigger or an IssueOps re-entry

    runs-on: ubuntu-24.04

      mode: ${{ steps.route.outputs.mode }}
      issue_number: ${{ steps.route.outputs.issue_number }}
      iteration: ${{ steps.route.outputs.iteration }}

      - name: Route RASL entry point
        id: route
        uses: actions/github-script@v7
        with:
          script: |
            const event = context.eventName;

              core.setOutput('mode', 'scan');
              core.setOutput('issue_number', '');
              core.setOutput('iteration', '0');
            } else if (event === 'issues') {
              const labels = context.payload.issue.labels.map(l => l.name);
              if (labels.includes('rasl-auto-patch')) {
                const body = context.payload.issue.body || '';
                const iterMatch = body.match(/RASL_ITERATION:\s*(\d+)/);
                const iter = iterMatch ? parseInt(iterMatch[^1]) : 0;
                if (iter >= parseInt(process.env.RASL_MAX_DEPTH)) {
                  core.warning(`RASL max depth ${process.env.RASL_MAX_DEPTH} reached. Halting autopoietic loop.`);
                  core.setOutput('mode', 'halt');
                } else {
                  core.setOutput('mode', 'patch');
                  core.setOutput('issue_number', String(context.payload.issue.number));
                  core.setOutput('iteration', String(iter + 1));
                }
              } else {
                core.setOutput('mode', 'skip');
              }
            }

  # PHASE 1: CodeQL Security Scan
  security-scan:
    needs: [rasl-router]
    if: needs.rasl-router.outputs.mode == 'scan'
    runs-on: ubuntu-24.04

      - uses: actions/checkout@v4

      - name: Initialize CodeQL
        uses: github/codeql-action/init@v3

          languages: "javascript,python"

        uses: github/codeql-action/autobuild@v3

        uses: github/codeql-action/analyze@v3

          output: codeql-results

      - name: Parse SARIF and extract actionable vulnerabilities
        id: parse-sarif
        run: |
          python3 << 'EOF'
          import json, glob, os

          sarif_files = glob.glob('codeql-results/**/*.sarif', recursive=True)

              with open(sarif_path) as f:
                  sarif = json.load(f)
              for run in sarif.get('runs', []):
                  for result in run.get('results', []):
                      sev = result.get('properties', {}).get('problem.severity', 'warning')
                      if sev in ['error', 'warning']:
                          loc = result.get('locations', [{}])[^0]
                          phys = loc.get('physicalLocation', {})
                          vulnerabilities.append({
                              "rule_id": result.get('ruleId', 'unknown'),
                              "message": result.get('message', {}).get('text', ''),
                              "severity": sev,
                              "file": phys.get('artifactLocation', {}).get('uri', ''),
                              "line": phys.get('region', {}).get('startLine', 0),
                              "snippet": phys.get('contextRegion', {}).get('snippet', {}).get('text', '')
                          })

          with open('vulnerabilities.json', 'w') as f:
              json.dump(vulnerabilities, f, indent=2)
          print(f"Extracted {len(vulnerabilities)} actionable vulnerabilities")
          with open(os.environ['GITHUB_OUTPUT'], 'a') as gh:
              gh.write(f"vuln_count={len(vulnerabilities)}\n")
          EOF

      - name: Create RASL autopoietic issue for each critical vulnerability
        if: steps.parse-sarif.outputs.vuln_count != '0'
        uses: actions/github-script@v7

          script: |
            const fs = require('fs');
            const vulns = JSON.parse(fs.readFileSync('vulnerabilities.json'));

            for (const vuln of critical) {
              await github.rest.issues.create({
                owner: context.repo.owner,
                repo: context.repo.repo,
                title: `[RASL-AUTO] ${vuln.rule_id} in ${vuln.file}:${vuln.line}`,
                body: `## Autopoietic Security Patch Request\n\n` +
                  `**Rule**: \`${vuln.rule_id}\`\n` +
                  `**Severity**: ${vuln.severity}\n` +
                  `**File**: \`${vuln.file}\`\n` +
                  `**Line**: ${vuln.line}\n\n` +
                  `### Vulnerable Code Snippet\n\`\`\`\n${vuln.snippet}\n\`\`\`\n\n` +
                  `### Message\n${vuln.message}\n\n` +

                labels: ['rasl-auto-patch', 'security']
              });
            }

  # PHASE 2: LLM Patch Generation (IssueOps re-entry)
  llm-patch-generator:
    needs: [rasl-router]
    if: needs.rasl-router.outputs.mode == 'patch'
    runs-on: ubuntu-24.04
    steps:
      - uses: actions/checkout@v4

      - name: Fetch issue details and extract vulnerability context
        id: fetch-issue
        uses: actions/github-script@v7
        with:
          script: |
            const issue = await github.rest.issues.get({

          ```

          ```
          {file_content[:8000]}

          patch_text = message.content[^0].text
          with open('security_patch.diff', 'w') as f:
              f.write(patch_text)
          print(f"LLM patch generated: {len(patch_text)} chars")

      - name: Apply patch and create PR
        run: |

      - name: Create patch PR with RASL metadata
        uses: actions/github-script@v7
        with:
          script: |
            const branch = `${{ env.PATCH_BRANCH_PREFIX }}-${{ needs.rasl-router.outputs.issue_number }}-iter${{ needs.rasl-router.outputs.iteration }}`;

### Architecture 4: The Temporal Escrow Smart Contract (TESC)

**Pattern:** Temporal Hash Invariance as Decentralized Contract
**Lens Applied:** Emergence \& Weak Signals
**Claim:** By combining cron-triggered artifact hash validation with conditional `workflow_dispatch` gates, a GitHub repository can act as a "Temporal Escrow" — holding cryptographically-verified state invariant until an external condition is satisfied, implementing the semantics of a blockchain smart contract without any blockchain infrastructure.

This implements the **Temporal Escrow Hypothesis** from the DRP. The mechanism: an artifact stores the "escrow payload" (a signed JSON manifest), a cron job validates its SHA-256 hash every N hours (preventing tampering), and a `workflow_dispatch` with a cryptographic unlock token releases the escrow by creating a production deployment. The invariance guarantee comes from the artifact's immutable storage within the runner retention window [][].

**Boundary Conditions:**
            - Escrow window limited to artifact retention policy (max 400 days)
            - Cryptographic security level: HMAC-SHA256 (sufficient for internal use; not production cryptographic grade for financial contracts)
            - Condition oracle must be an externally-callable API endpoint; GitHub Actions cannot natively monitor arbitrary off-chain state

```yaml
# +++PetzoldSequence[DRP-GHA-TESC-004]
# Architecture: Temporal Escrow Smart Contract
# Topology Class: Temporal Anchor × State Reader × Conditional Emission
# Escrow Semantics: HOLD state invariant until off-chain oracle confirms release condition

name: "TESC — Temporal Escrow Smart Contract"

on:
  workflow_dispatch:
    inputs:
      operation:
        description: "DEPOSIT | VERIFY | RELEASE"
        required: true
        type: choice
        options: [DEPOSIT, VERIFY, RELEASE]
      payload_b64:
        description: "Base64-encoded escrow payload JSON (for DEPOSIT)"
        type: string
      unlock_token:
        description: "HMAC-SHA256 unlock token (for RELEASE)"
        type: string
      oracle_url:
        description: "External condition oracle endpoint URL (for RELEASE)"
        type: string
  schedule:
    - cron: "0 */6 * * *"  # Integrity heartbeat every 6 hours

env:
  ESCROW_ARTIFACT_NAME: "tesc-escrow-payload-v1"
  ESCROW_HMAC_KEY_SECRET: "TESC_SIGNING_KEY"

jobs:
  tesc-operation:
    runs-on: ubuntu-24.04
    steps:
      - name: Route TESC operation
        id: route
        run: |
          if [ "${{ github.event_name }}" == "schedule" ]; then
            echo "op=VERIFY" >> $GITHUB_OUTPUT
          else
            echo "op=${{ inputs.operation }}" >> $GITHUB_OUTPUT
          fi

      # === DEPOSIT: Lock payload into temporal escrow ===
      - name: "[DEPOSIT] Create and sign escrow payload"
        if: steps.route.outputs.op == 'DEPOSIT'
        run: |
          python3 << 'EOF'
          import json, hashlib, hmac, base64, os, datetime

          payload_b64 = "${{ inputs.payload_b64 }}"
          payload_bytes = base64.b64decode(payload_b64)
          payload = json.loads(payload_bytes)

          signing_key = os.environ.get('TESC_SIGNING_KEY', 'default-dev-key-replace-in-prod').encode()

          escrow_manifest = {
              "version": "TESC-2026",
              "deposited_at": datetime.datetime.utcnow().isoformat() + "Z",
              "depositor": "${{ github.actor }}",
              "payload": payload,
              "status": "LOCKED",
              "integrity": None,
              "unlock_conditions": {
                  "requires_oracle": bool("${{ inputs.oracle_url }}"),
                  "oracle_url": "${{ inputs.oracle_url }}",
                  "max_lock_duration_hours": 720,
              }
          }

          # HMAC-SHA256 integrity seal
          manifest_bytes = json.dumps({k: v for k, v in escrow_manifest.items() if k != 'integrity'}, sort_keys=True).encode()
          seal = hmac.new(signing_key, manifest_bytes, hashlib.sha256).hexdigest()
          escrow_manifest['integrity'] = seal

          with open('escrow_manifest.json', 'w') as f:
              json.dump(escrow_manifest, f, indent=2)

          print(f"Escrow sealed. HMAC: {seal[:16]}...")
          print(f"Status: LOCKED | Depositor: {escrow_manifest['depositor']}")
          EOF
        env:
          TESC_SIGNING_KEY: ${{ secrets.TESC_SIGNING_KEY }}

      - name: "[DEPOSIT] Upload escrow to temporal artifact store"
        if: steps.route.outputs.op == 'DEPOSIT'
        uses: actions/upload-artifact@v4
        with:
          name: ${{ env.ESCROW_ARTIFACT_NAME }}
          path: escrow_manifest.json
          retention-days: 30

      # === VERIFY: Heartbeat integrity check (cron-triggered) ===
      - name: "[VERIFY] Download escrow artifact"
        if: steps.route.outputs.op == 'VERIFY'
        uses: actions/download-artifact@v4
        with:
          name: ${{ env.ESCROW_ARTIFACT_NAME }}
          path: ./escrow
        continue-on-error: true

      - name: "[VERIFY] Validate temporal invariance"
        if: steps.route.outputs.op == 'VERIFY'
        run: |
          python3 << 'EOF'
          import json, hashlib, hmac, os, sys
          from pathlib import Path

          escrow_path = Path('./escrow/escrow_manifest.json')
          if not escrow_path.exists():
              print("No active escrow. Heartbeat skipped.")
              sys.exit(0)

          stored_seal = manifest.pop('integrity')
          manifest_bytes = json.dumps(manifest, sort_keys=True).encode()
          computed_seal = hmac.new(signing_key, manifest_bytes, hashlib.sha256).hexdigest()

          if stored_seal != computed_seal:
              print(f"::error::TESC INTEGRITY VIOLATION DETECTED! Escrow payload has been tampered.")
              print(f"Expected: {stored_seal[:16]}... | Got: {computed_seal[:16]}...")
              sys.exit(1)

          print(f"Temporal invariance confirmed. Seal: {stored_seal[:16]}...")

          EOF

      - name: "[RELEASE] Verify oracle condition and unlock token"

                  condition_met = oracle_response.get('release_authorized', False)
                  print(f"Oracle response: {oracle_response}")
              except Exception as e:
                  print(f"::error::Oracle unreachable: {e}")
                  sys.exit(1)
          else:
              # Direct token-only release
              condition_met = True

          if not condition_met:
              print("::error::Oracle condition NOT met. Escrow remains locked.")
              sys.exit(1)

          # Validate HMAC unlock token against escrow content
          expected_token = hmac.new(signing_key, b"RELEASE_AUTHORIZED", hashlib.sha256).hexdigest()
          if not hmac.compare_digest(unlock_token.strip(), expected_token):
              print("::error::Invalid unlock token. Escrow remains locked.")
              sys.exit(1)

          print("RELEASE AUTHORIZED: Oracle condition met + token verified.")
          with open(os.environ['GITHUB_OUTPUT'], 'a') as gh:
              gh.write("release_authorized=true\n")
          EOF
        env:
          TESC_SIGNING_KEY: ${{ secrets.TESC_SIGNING_KEY }}

      - name: "[RELEASE] Execute escrowed deployment payload"
        if: steps.route.outputs.op == 'RELEASE' && steps.verify-oracle.outputs.release_authorized == 'true'
        uses: actions/download-artifact@v4
        with:
          name: ${{ env.ESCROW_ARTIFACT_NAME }}
          path: ./escrow-release

      - name: "[RELEASE] Trigger deployment from escrow payload"
        if: steps.route.outputs.op == 'RELEASE'
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            try {
              const manifest = JSON.parse(fs.readFileSync('./escrow-release/escrow_manifest.json'));
              await github.rest.repos.createDeployment({
                owner: context.repo.owner,
                repo: context.repo.repo,
                ref: manifest.payload.target_ref || 'main',
                environment: manifest.payload.environment || 'production',
                description: `TESC Release: ${manifest.deposited_at}`,
                auto_merge: false,
                required_contexts: []
              });
              console.log('Temporal escrow released. Deployment initiated.');
            } catch(e) {
              core.warning(`Deployment payload missing or invalid: ${e.message}`);
            }

### Architecture 5: The Metacognitive Drift Sentinel (MDS)

**Pattern:** Self-Aware Repository Topology Maintenance
**Lens Applied:** Emergence \& Weak Signals + Boundary Spanning
**Claim:** By composing dependency graph analysis actions, semantic diff actions, and an LLM critique engine, a repository can continuously compute its own "cognitive drift" — the divergence between its current implementation topology and its declared architectural intent — and file precise, actionable issue reports without human observation.

This is the most architecturally novel workflow because it treats the **repository as a cognitive subject** rather than a passive code container. The workflow encodes the repository's `ARCHITECTURE.md` (or equivalent) as the "intent state," computes structural metrics of the actual codebase (coupling, cohesion, dependency depth), compares them via LLM semantic analysis, and reports drift as quantified "Projection Tax" — the cost of the gap between declared and actual architecture [][].

**Boundary Conditions:**
            - Requires a machine-readable `ARCHITECTURE.md` or `docs/architecture.json` file as ground truth
            - LLM token costs increase with codebase size; gate large repos with path filters
            - Recursive drift: if MDS itself is a repository module, it must exclude itself from its own drift analysis to prevent infinite regress

```yaml
# +++PetzoldSequence[DRP-GHA-MDS-005]
# Architecture: Metacognitive Drift Sentinel
# Topology Class: State Reader (codebase) × Transformation (LLM) × Emission (Issue/PR)
# Metacognition: Repository observes and critiques its own structural coherence

name: "MDS — Metacognitive Drift Sentinel"

on:
  schedule:
    - cron: "0 4 * * 0"  # Weekly metacognitive audit
  push:
    paths:
      - "src/**"
      - "lib/**"
      - "*.toml"
      - "*.json"
      - "package.json"
      - "Cargo.toml"
      - "pyproject.toml"
    branches: [main]

permissions:
  contents: read
  issues: write

env:
  DRIFT_THRESHOLD: "0.25"  # Normalized drift score above which an issue is filed
  LLM_MODEL: "claude-opus-4-6"

jobs:
  # PHASE 1: Structural Topology Extraction
  extract-topology:
    runs-on: ubuntu-24.04
    outputs:
      topology_hash: ${{ steps.hash.outputs.hash }}
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # Full history for drift velocity calculation

      - name: Install topology analysis tools (polyglot)
        run: |
          pip install lizard radon networkx graphviz 2>/dev/null || true
          npm install -g madge 2>/dev/null || true
          cargo install cargo-depgraph 2>/dev/null || true
          sudo apt-get install -y cloc

      - name: Compute structural topology metrics
        run: |
          python3 << 'EOF'
          import subprocess, json, os, re
          from pathlib import Path

          topology = {
              "computed_at": os.popen("date -u +%Y%m%dT%H%M%SZ").read().strip(),
              "commit": os.popen("git rev-parse HEAD").read().strip(),
              "language_distribution": {},
              "complexity_metrics": {},
              "coupling_metrics": {},
              "dependency_depth": {},
              "architectural_modules": []
          }

          # Language distribution via cloc
          cloc_result = subprocess.run(['cloc', '--json', '.', '--exclude-dir=node_modules,.git,target,dist'],
                                       capture_output=True, text=True)
          if cloc_result.returncode == 0:
              try:
                  cloc_data = json.loads(cloc_result.stdout)
                  topology['language_distribution'] = {
                      k: {"files": v.get("nFiles", 0), "code_lines": v.get("code", 0)}
                      for k, v in cloc_data.items()
                      if k not in ('header', 'SUM')
                  }
              except: pass

          # Cyclomatic complexity via lizard (Python/JS/C/Go/Rust)
          try:
              import lizard
              results = lizard.analyze(['.'])
              functions = [(f.name, f.cyclomatic_complexity, f.filename)
                          for r in results for f in r.function_list
                          if f.cyclomatic_complexity > 5]
              topology['complexity_metrics'] = {
                  "high_complexity_functions": [{"name": n, "cc": cc, "file": fn} for n, cc, fn in functions[:20]],
                  "function_count": sum(len(r.function_list) for r in lizard.analyze(['.']))
              }
          except Exception as e:
              topology['complexity_metrics']['error'] = str(e)

          # Module boundary detection from directory structure
          src_dirs = [str(p) for p in Path('.').glob('src/*') if p.is_dir()]
          lib_dirs = [str(p) for p in Path('.').glob('lib/*') if p.is_dir()]
          topology['architectural_modules'] = src_dirs + lib_dirs

          # Dependency depth from package manifests
          for manifest in ['package.json', 'Cargo.toml', 'pyproject.toml', 'go.mod']:
              if Path(manifest).exists():
                  content = Path(manifest).read_text()
                  topology['dependency_depth'][manifest] = content.count('\n')

          with open('codebase_topology.json', 'w') as f:
              json.dump(topology, f, indent=2)
          print(f"Topology extracted: {len(topology['architectural_modules'])} modules detected")
          EOF

      - name: Hash topology for drift velocity tracking
        id: hash
        run: |
          HASH=$(sha256sum codebase_topology.json | cut -d' ' -f1)
          echo "hash=$HASH" >> $GITHUB_OUTPUT

      - uses: actions/upload-artifact@v4
        with:
          name: codebase-topology
          path: codebase_topology.json
          retention-days: 30

  # PHASE 2: Intent State Extraction (ARCHITECTURE.md → structured intent)
  extract-intent:
    runs-on: ubuntu-24.04
    steps:
      - uses: actions/checkout@v4

      - name: Parse architectural intent document
        run: |
          python3 << 'EOF'
          import os, json
          from pathlib import Path

          # Search for architectural intent documents
          intent_sources = ['ARCHITECTURE.md', 'docs/ARCHITECTURE.md', 'docs/architecture.json',
                           'ADR.md', 'docs/adr/', 'DESIGN.md']
          intent_content = []

          for source in intent_sources:
              p = Path(source)
              if p.exists():
                  if p.is_file():
                      intent_content.append({"source": source, "content": p.read_text()[:10000]})
                  elif p.is_dir():
                      for f in p.glob('*.md'):
                          intent_content.append({"source": str(f), "content": f.read_text()[:5000]})

          if not intent_content:
              # Generate synthetic intent from README
              readme = Path('README.md')
              if readme.exists():
                  intent_content = [{"source": "README.md", "content": readme.read_text()[:8000]}]
              else:
                  intent_content = [{"source": "synthetic", "content": "No architectural intent document found. Using empty baseline."}]

          with open('architectural_intent.json', 'w') as f:
              json.dump({"sources": intent_content, "intent_count": len(intent_content)}, f, indent=2)
          print(f"Intent extracted from {len(intent_content)} source(s)")
          EOF

      - uses: actions/upload-artifact@v4
        with:
          name: architectural-intent
          path: architectural_intent.json
          retention-days: 1

  # PHASE 3: LLM Drift Computation
  compute-drift:
    needs: [extract-topology, extract-intent]
    runs-on: ubuntu-24.04
    outputs:
      drift_score: ${{ steps.drift.outputs.drift_score }}
      drift_report: ${{ steps.drift.outputs.drift_report_path }}
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: codebase-topology
          path: ./artifacts
      - uses: actions/download-artifact@v4
        with:
          name: architectural-intent
          path: ./artifacts

      - name: Compute metacognitive drift via LLM analysis
        id: drift
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          python3 << 'EOF'
          import anthropic, json, os, re

          with open('./artifacts/codebase_topology.json') as f:
              topology = json.load(f)
          with open('./artifacts/architectural_intent.json') as f:
              intent = json.load(f)

          client = anthropic.Anthropic(api_key=os.environ['ANTHROPIC_API_KEY'])

          intent_text = "\n\n".join([f"### {s['source']}\n{s['content']}" for s in intent['sources']])
          topology_summary = json.dumps({
              "languages": topology.get('language_distribution', {}),
              "high_complexity_functions": topology.get('complexity_metrics', {}).get('high_complexity_functions', [])[:5],
              "modules": topology.get('architectural_modules', []),
              "commit": topology.get('commit', '')
          }, indent=2)

          message = client.messages.create(
              model="claude-opus-4-6",
              max_tokens=3000,
              messages=[{
                  "role": "user",
                  "content": f"""You are a metacognitive architecture sentinel. Analyze the drift between declared architectural intent and actual codebase topology.

          ## DECLARED ARCHITECTURAL INTENT
          {intent_text[:6000]}

          ## ACTUAL CODEBASE TOPOLOGY (extracted metrics)
          ```json
          {topology_summary}
          ```

          Provide your analysis as JSON with these exact fields:
          {{
            "drift_score": <float 0.0-1.0, where 0=perfect alignment, 1=complete divergence>,
            "projection_tax": <string: estimated cost of this drift in developer-days>,
            "drift_dimensions": [
              {{"dimension": "coupling|cohesion|naming|responsibility|dependency", "score": <0-1>, "evidence": "specific observation"}}
            ],
            "critical_violations": ["specific violation 1", "specific violation 2"],
            "recommended_actions": [
              {{"priority": "HIGH|MEDIUM|LOW", "action": "specific refactoring action", "file_hint": "path if known"}}
            ],
            "architectural_health_grade": "A|B|C|D|F"
          }}

          Return ONLY valid JSON. No markdown, no explanation."""
              }]
          )

          try:
              drift_analysis = json.loads(message.content[^0].text)
          except json.JSONDecodeError:
              # Fallback structure if LLM returns malformed JSON
              drift_analysis = {
                  "drift_score": 0.5,
                  "projection_tax": "Unknown",
                  "drift_dimensions": [],
                  "critical_violations": ["LLM response parsing error"],
                  "recommended_actions": [],
                  "architectural_health_grade": "C"
              }

          drift_analysis['topology_hash'] = "${{ needs.extract-topology.outputs.topology_hash }}"
          drift_analysis['commit'] = topology.get('commit', '')

          with open('drift_report.json', 'w') as f:
              json.dump(drift_analysis, f, indent=2)

          score = drift_analysis.get('drift_score', 0)
          print(f"Drift score: {score:.2f} | Grade: {drift_analysis.get('architectural_health_grade')}")
          print(f"Projection tax: {drift_analysis.get('projection_tax')}")

          with open(os.environ['GITHUB_OUTPUT'], 'a') as gh:
              gh.write(f"drift_score={score}\n")
              gh.write("drift_report_path=drift_report.json\n")
          EOF

      - uses: actions/upload-artifact@v4
        with:
          name: drift-report
          path: drift_report.json
          retention-days: 30

  # PHASE 4: Conditional Issue Filing Based on Drift Threshold
  file-drift-issue:
    needs: [compute-drift]
    runs-on: ubuntu-24.04
    if: |
      needs.compute-drift.outputs.drift_score != '' &&
      (needs.compute-drift.outputs.drift_score > 0.25)
    steps:
      - uses: actions/download-artifact@v4
        with:
          name: drift-report
          path: ./drift

      - name: File metacognitive drift issue
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const report = JSON.parse(fs.readFileSync('./drift/drift_report.json'));

            const violationsList = report.critical_violations.map(v => `- ${v}`).join('\n');
            const actionsList = report.recommended_actions.map(a =>
              `- **[${a.priority}]** ${a.action}${a.file_hint ? ` (\`${a.file_hint}\`)` : ''}`
            ).join('\n');
            const dimensionTable = report.drift_dimensions.map(d =>
              `| ${d.dimension} | ${(d.score * 100).toFixed(0)}% | ${d.evidence} |`
            ).join('\n');

            const body = `## 🧠 Metacognitive Drift Report

            **Drift Score**: ${(report.drift_score * 100).toFixed(1)}% | **Health Grade**: ${report.architectural_health_grade} | **Projection Tax**: ${report.projection_tax}

            > This issue was auto-generated by the MDS (Metacognitive Drift Sentinel). It represents the computed divergence between declared architectural intent and actual codebase topology.

            ### Drift Dimensions
            | Dimension | Drift % | Evidence |
            |---|---|---|
            ${dimensionTable}

            ### Critical Violations
            ${violationsList}

            ### Recommended Actions
            ${actionsList}

            ---
            *Commit*: \`${report.commit}\` | *Topology Hash*: \`${report.topology_hash?.slice(0,16)}\``;

            // Check for existing open drift issue to avoid duplicates
            const existing = await github.rest.issues.listForRepo({
              owner: context.repo.owner,
              repo: context.repo.repo,
              labels: 'mds-drift-report',
              state: 'open'
            });

            if (existing.data.length > 0) {
              // Update existing issue
              await github.rest.issues.createComment({
                owner: context.repo.owner,
                repo: context.repo.repo,
                issue_number: existing.data[^0].number,
                body: `## Updated Drift Analysis\n\n${body}`
              });
            } else {
              // Create new drift issue
              await github.rest.issues.create({
                owner: context.repo.owner,
                repo: context.repo.repo,
                title: `[MDS] Architectural Drift Alert — Score: ${(report.drift_score * 100).toFixed(1)}% (Grade: ${report.architectural_health_grade})`,
                body,
                labels: ['mds-drift-report', 'architecture', 'technical-debt']
              });
            }
```
